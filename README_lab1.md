## 1. Математический фундамент нейронных сетей

### 1.1 Линейный слой и матричное представление

В основе любого полносвязного (Dense) слоя нейронной сети лежит операция аффинного преобразования входного пространства признаков. Для одиночного обучающего примера, представленного вектором-строкой $x$, математическая модель преактивации (взвешенной суммы) имеет вид:

$$z = \sum_{i=1}^{n} x_i w_i + b = x \cdot w + b$$

При переходе к пакетной обработке данных (batch processing) операции переводятся в матричную форму. Пусть $X$ — матрица объект-признак размера $(m \times n)$, где $m$ — объем выборки (размер батча), а $n$ — количество входных признаков. Если слой содержит $h$ нейронов, то матрица весов $W$ имеет размерность $(n \times h)$, а вектор смещений $b$ — размерность $(1 \times h)$. Матричное уравнение принимает вид:

$$Z = X \cdot W + b$$

В коде этот этап реализуется с помощью оператора матричного умножения `@` (вызывающего функцию `np.dot` под капотом):

```python
self.z = X @ self.w + self.b

```

Прибавление вектора смещений $b$ к матрице $XW$ автоматически использует механизм **broadcasting** (трансляции) в `numpy`: вектор строки $(1 \times h)$ копируется по вертикали $m$ раз для соответствия размерности матрицы $(m \times h)$.

### 1.2 Функции активации и их производные

Линейные операции сами по себе не позволяют сети строить нелинейные разделяющие поверхности. Композиция любого количества линейных слоев математически эквивалентна одному линейному слою:

$$Z_2 = (X W_1 + b_1) W_2 + b_2 = X (W_1 W_2) + (b_1 W_2 + b_2) = X W_{eff} + b_{eff}$$

Для обучения сложных зависимостей в архитектуру вводятся нелинейные функции активации.

#### Сигмоида (Sigmoid)

Используется в скрытых слоях и на выходном слое для задач бинарной классификации:

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

Функция отображает всю вещественную прямую в открытый интервал $(0, 1)$, что позволяет интерпретировать выходной сигнал как апостериорную вероятность принадлежности объекта к целевому классу.

Важным свойством сигмоиды является то, что её производная выражается через её же значение. Это существенно экономит вычислительные ресурсы при обратном проходе:

$$\frac{d\sigma(z)}{dz} = \sigma(z) \cdot (1 - \sigma(z))$$

При реализации экспоненты на компьютере возникает проблема численной нестабильности (**overflow**): при больших отрицательных значениях $z$ (например, $z = -1000$), величина $e^{-z}$ превращается в компьютерную бесконечность. Для предотвращения этого применяется операция жесткого ограничения аргумента (`np.clip`):

```python
def sigmoid(z):
    return 1.0 / (1.0 + np.exp(-np.clip(z, -500, 500)))

```

#### Софтмакс (Softmax)

Для многоклассовой классификации (когда количество классов $K > 2$) сигмоида не подходит, так как выходы сети для одного объекта должны быть взаимоисключающими вероятностями, сумма которых равна $1$. Используется функция Softmax:

$$\text{softmax}(z_i) = \frac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}}$$

Численная нестабильность здесь проявляется при больших положительных $z$: экспонента от $800$ вызывает переполнение типа `float64`. Для борьбы с этим применяется инвариантность Softmax к сдвигу: если из каждого элемента вектора вычесть одну и ту же константу, значение функции не изменится. В качестве константы выбирают максимум строки:

$$\text{softmax}(z_i) = \frac{e^{z_i - \max(Z)}}{\sum_{j=1}^{K} e^{z_j - \max(Z)}}$$

```python
def softmax(z):
    z_shifted = z - np.max(z, axis=1, keepdims=True)
    exp_z = np.exp(z_shifted)
    return exp_z / np.sum(exp_z, axis=1, keepdims=True)

```

### 1.3 Функции потерь (Loss Functions)

#### Бинарная кросс-энтропия (Binary Cross-Entropy Loss)

Применяется при обучении одиночного нейрона для разделения двух классов ($y_i \in \{0, 1\}$):

$$L = -\frac{1}{m} \sum_{i=1}^{m} \left[ y_i \ln(a_i) + (1 - y_i) \ln(1 - a_i) \right]$$

Где $a_i = \sigma(z_i)$ — предсказание сети. Если $a_i \to 0$ при истинном $y_i = 1$, значение логарифма стремится к $-\infty$, а функция потерь штрафует сеть огромным значением ошибки. Для предотвращения вычисления $\ln(0)$ предсказания клипируются значением $\epsilon = 10^{-15}$:

```python
eps = 1e-15
a = np.clip(self.a, eps, 1 - eps)
loss = -np.mean(y * np.log(a) + (1 - y) * np.log(1 - a))

```

#### Многоклассовая кросс-энтропия (Categorical Cross-Entropy Loss)

Применяется для $K$ классов, где метки $y$ закодированы с помощью One-Hot Encoding:

$$L = -\frac{1}{m} \sum_{i=1}^{m} \sum_{j=1}^{K} y_{ij} \ln(a_{ij})$$

```python
a = np.clip(self.activations[-1], 1e-15, 1 - 1e-15)
loss = -np.mean(np.sum(y_true * np.log(a), axis=1))

```

## 2. Теория обратного распространения ошибки (Backpropagation)

Обучение нейросети — это задача минимизации функции потерь $L(W, b)$ методом градиентного спуска. Нам необходимо вычислить частные производные $\frac{\partial L}{\partial W}$ и $\frac{\partial L}{\partial b}$ для каждого слоя.

### 2.1 Математический вывод для выходного слоя

Рассмотрим связку: функция потерь (Cross-Entropy) + финальная активация (Sigmoid или Softmax). Одно из красивейших аналитических свойств заключается в том, что при поиске производной функции потерь по входному значению преактивации $z$ сложные дроби сокращаются.

Применим цепное правило (Chain Rule):

$$\frac{\partial L}{\partial z} = \frac{\partial L}{\partial a} \cdot \frac{\partial a}{\partial z}$$

Для бинарной кросс-энтропии одной точки:

$$\frac{\partial L}{\partial a} = -\left( \frac{y}{a} - \frac{1-y}{1-a} \right) = \frac{a - y}{a(1-a)}$$

Производная сигмоиды:

$$\frac{\partial a}{\partial z} = a(1-a)$$

Перемножая их, получаем градиент ошибки выходного слоя (обозначаемый как $\delta$):

$$\delta = \frac{\partial L}{\partial z} = \frac{a - y}{a(1-a)} \cdot a(1-a) = a - y$$

Для связки Categorical Cross-Entropy + Softmax аналитический вывод дает точно такой же результат: вектор ошибки равен разности между вектором предсказанных вероятностей и вектором истинных One-Hot меток:

$$\delta = \hat{Y} - Y$$

В коде это элементарная операция:

```python
delta = self.activations[-1] - y_true

```

### 2.2 Прохождение ошибки через скрытые слои

Пусть у нас есть скрытый слой $l$, за которым следует слой $l+1$. Нам известен градиент ошибки на слое $l+1$: $\delta^{l+1} = \frac{\partial L}{\partial Z^{l+1}}$. Найдем ошибку на слое $l$:

$$\delta^l = \frac{\partial L}{\partial Z^l} = \frac{\partial L}{\partial Z^{l+1}} \cdot \frac{\partial Z^{l+1}}{\partial A^l} \cdot \frac{\partial A^l}{\partial Z^l}$$

Поскольку $Z^{l+1} = A^l \cdot W^{l+1} + b^{l+1}$, частная производная $\frac{\partial Z^{l+1}}{\partial A^l} = (W^{l+1})^T$.
Частная производная активации по преактивации — это локальная производная функции активации скрытого слоя $\sigma'(Z^l)$.

В матричной форме для пакета данных уравнение имеет вид:

$$\delta^l = \left( \delta^{l+1} \cdot (W^{l+1})^T \right) \odot \sigma'(A^l)$$

Где $\odot$ — покомпонентное умножение матриц (Hadamard product). Обратите внимание, что в коде функция `sigmoid_derivative` принимает уже активированное значение $A^l$, так как $\sigma'(z) = A(1-A)$.

```python
delta_prev = (delta @ self.W[l].T) * sigmoid_derivative(a_prev)

```

### 2.3 Вычисление градиентов по параметрам

Зная ошибку $\delta^l$ на любом слое, мы можем рассчитать, как изменение весов $W^l$ и смещений $b^l$ этого слоя повлияет на финальную ошибку сети:

$$\frac{\partial L}{\partial W^l} = (A^{l-1})^T \cdot \delta^l$$

$$\frac{\partial L}{\partial b^l} = \sum_{i=1}^{m} \delta^l_i$$

Для усреднения градиента по батчу мы делим матричное произведение на количество объектов $m$:

```python
dW = (a_prev.T @ delta) / m
db = np.mean(delta, axis=0, keepdims=True)

```

### 2.4 Обновление параметров (Стохастический градиентный спуск — SGD)

Получив матрицы градиентов, параметры корректируются в сторону, противоположную вектору градиента, с шагом, регулируемым гиперпараметром `lr` ($\eta$ — learning rate):

$$W^l = W^l - \eta \cdot dW^l$$

$$b^l = b^l - \eta \cdot db^l$$

```python
self.W[l] -= self.lr * dW
self.b[l] -= self.lr * db

```

## 3. Проблема инициализации весов: Метод Ксавье (Xavier / Glorot)

Если инициализировать все веса сети нулями, то на этапе прямого прохода все нейроны скрытого слоя получат одинаковые значения преактиваций. Во время обратного прохода градиенты для всех нейронов слоя окажутся идентичными. Сеть потеряет свойство **разрушения симметрии** (symmetry breaking) — независимо от количества скрытых нейронов, они все будут обучаться одинаково, превращаясь по сути в один нейрон.

Если инициализировать веса из стандартного нормального распределения $\mathcal{N}(0, 1)$, то в глубоких сетях с большим количеством входов дисперсия преактиваций $Z$ будет расти пропорционально числу входных связей (`fan_in`). Это приведет к тому, что значения $Z$ будут уходить далеко в области насыщения сигмоиды (большие положительные или отрицательные числа), где её производная практически равна нулю. Возникнет **проблема затухания градиентов** (vanishing gradients), и сеть перестанет обучаться.

Для решения этой проблемы используется инициализация Ксавье. Веса выбираются из нормального распределения с нулевым математическим ожиданием и масштабируемой дисперсией:

$$\text{Var}(W) = \frac{2}{\text{fan\_in} + \text{fan\_out}}$$

В нашей лабораторной работе используется классический вариант масштабирования:

```python
scale = np.sqrt(2.0 / fan_in)
self.W.append(np.random.randn(fan_in, fan_out) * scale)

```

Это удерживает дисперсию сигналов на каждом слое примерно одинаковой, обеспечивая стабильный поток градиентов как вперед, так и назад.

## 4. Подробный разбор программной реализации

### 4.1 Класс `SingleNeuron`

Этот класс реализует базовую модель изолированного искусственного нейрона для бинарной классификации.

```python
class SingleNeuron:
    def __init__(self, n_features: int, lr: float = 0.1):
        self.lr = lr
        scale = np.sqrt(2.0 / n_features)
        self.w = np.random.randn(n_features, 1) * scale
        self.b = np.zeros((1, 1))
        self.loss_history = []

    def forward(self, X):
        self.X = X
        self.z = X @ self.w + self.b       # Размерность: (m, n) x (n, 1) -> (m, 1)
        self.a = sigmoid(self.z)           # Размерность: (m, 1)
        return self.a

    def compute_loss(self, y):
        m = y.shape[0]
        eps = 1e-15
        a = np.clip(self.a, eps, 1 - eps)
        loss = -np.mean(y * np.log(a) + (1 - y) * np.log(1 - a))
        return loss

    def backward(self, y):
        m = y.shape[0]
        delta = self.a - y                 # Выходная ошибка (m, 1)
        dw = (self.X.T @ delta) / m        # Градиент весов: (n, m) x (m, 1) -> (n, 1)
        db = np.mean(delta)                # Скалярное смещение
        
        # Шаг градиентного спуска
        self.w -= self.lr * dw
        self.b -= self.lr * db

    def fit(self, X, y, epochs: int = 1000, verbose: int = 100):
        y = y.reshape(-1, 1)
        self.loss_history = []
        for epoch in range(1, epochs + 1):
            self.forward(X)
            loss = self.compute_loss(y)
            self.loss_history.append(loss)
            self.backward(y)
            if verbose and epoch % verbose == 0:
                print(f'Epoch {epoch:5d}/{epochs} | Loss: {loss:.6f}')
        return self

```

**Анализ механики вычислений:**

* На этапе обучения (`fit`) метод `forward` кэширует входную матрицу `self.X` в объект класса. Это необходимо, так как во время работы `backward` нам потребуется исходный входной сигнал для вычисления градиента весов: `self.X.T @ delta`.
* Метод `predict` производит бинаризацию вещественных вероятностей по жесткому порогу (treshold): объекты со значением $\ge 0.5$ относятся к классу $1$, остальные — к классу $0$.

### 4.2 Класс `MLP` (Многослойный перцептрон)

Класс спроектирован динамическим: архитектура сети задается списком целых чисел `layer_sizes`. Например, инициализация `layer_sizes=[4, 10, 10, 3]` создаст сеть с 4 входами, двумя скрытыми слоями по 10 нейронов и выходным слоем на 3 нейрона.

```python
class MLP:
    def __init__(self, layer_sizes: list, lr: float = 0.01, output_mode: str = 'softmax'):
        self.lr = lr
        self.output_mode = output_mode
        self.n_layers = len(layer_sizes) - 1

        self.W = []
        self.b = []
        for i in range(self.n_layers):
            fan_in  = layer_sizes[i]
            fan_out = layer_sizes[i + 1]
            scale = np.sqrt(2.0 / fan_in)
            self.W.append(np.random.randn(fan_in, fan_out) * scale)
            self.b.append(np.zeros((1, fan_out)))

        self.loss_history = []

    def forward(self, X):
        # Очищаем и инициализируем список активаций. a[0] — это входной вектор X.
        self.activations = [X]
        a = X
        for l in range(self.n_layers):
            z = a @ self.W[l] + self.b[l]
            if l == self.n_layers - 1:
                # Финальный слой имеет специфическую активацию
                a = softmax(z) if self.output_mode == 'softmax' else sigmoid(z)
            else:
                # Скрытые слои активируются сигмоидой
                a = sigmoid(z)
            self.activations.append(a)
        return self.activations[-1]

```

**Разбор обратного прохода в MLP:**
Ключевая сложность заключается в итерации по слоям в обратном порядке при помощи `reversed(range(self.n_layers))`.

```python
    def backward(self, y_true):
        m = y_true.shape[0]
        # Градиент на финальном слое
        delta = self.activations[-1] - y_true   

        for l in reversed(range(self.n_layers)):
            a_prev = self.activations[l]         # Активации предыдущего слоя
            dW = (a_prev.T @ delta) / m          # Градиент весов текущего слоя
            db = np.mean(delta, axis=0, keepdims=True)
            
            if l > 0:
                # Расчет ошибки для предыдущего слоя перед обновлением текущих весов
                delta_prev = (delta @ self.W[l].T) * sigmoid_derivative(a_prev)
            
            # Обновление параметров слоя l
            self.W[l] -= self.lr * dW
            self.b[l] -= self.lr * db
            
            if l > 0:
                delta = delta_prev

```

*Важная деталь:* Расчет переменной `delta_prev` должен выполняться **до** модификации матриц весов текущего слоя `self.W[l] -= self.lr * dW`. Если сначала обновить веса, то проброс ошибки назад произойдет по уже измененным весовым коэффициентам, что нарушит математическую строгость алгоритма и приведет к расходимости обучения.

## 5. Подготовка данных Iris и предобработка

Датасет Iris содержит 150 материальных объектов, равномерно распределенных по 3 классам (Setosa, Versicolor, Virginica). Каждый объект описывается 4 признаками: длины и ширины чашелистиков (sepal) и лепестков (petal).

### 5.1 Необходимость стандартизации (Z-score normalization)

Нейронные сети крайне чувствительны к масштабу входных данных. Если один признак варьируется в диапазоне $[0.1, 0.5]$, а второй — $[100, 5000]$, то градиенты по весам первого признака будут ничтожно малы, а по весам второго признака — огромны. Линии уровня функции потерь превратятся в сильно вытянутые эллипсы, и градиентный спуск начнет осциллировать (совершать зигзагообразные движения), замедляя сходимость.

Для устранения этой проблемы применяется класс `StandardScaler`:

$$x_{new} = \frac{x - \mu}{\sigma}$$

Где $\mu$ — среднее значение признака, а $\sigma$ — среднеквадратичное отклонение. После стандартизации каждый признак имеет математическое ожидание $0$ и дисперсию $1$.

*Важное правило:* Расчет параметров масштабирования (`fit`) выполняется **строго на тренировочной выборке**, после чего тестовая выборка только трансформируется (`transform`). Сеть не должна получать информацию о статистиках тестового набора данных до этапа проверки.

```python
scaler_mlp = StandardScaler()
X_mlp = scaler_mlp.fit_transform(X_full)

```

### 5.2 Пакетное кодирование меток (One-Hot Encoding)

Для работы выходного слоя с функцией активации `softmax` целочисленные метки классов $y \in \{0, 1, 2\}$ переводятся в ортогональные бинарные векторы. Функция `one_hot` формирует разреженную матрицу:

$$\text{Класс } 0 \to [1, 0, 0], \quad \text{Класс } 1 \to [0, 1, 0], \quad \text{Класс } 2 \to [0, 0, 1]$$

```python
def one_hot(y, n_classes):
    m = len(y)
    oh = np.zeros((m, n_classes))
    oh[np.arange(m), y] = 1
    return oh

```

## 6. Анализ экспериментов и визуализация

### 6.1 Одиночный нейрон на линейно разделимой подвыборке

В первом эксперименте рассматривается подвыборка, содержащая только два класса: *Setosa* и *Versicolor*, и только два признака (*sepal length*, *sepal width*). Из ботаники и разведочного анализа данных известно, что класс *Setosa* расположен изолированно от двух других видов.

* **Параметры обучения:** `lr=0.5`, `epochs=2000`.
* **Динамика потерь:** График `BCE Loss` плавно и монотонно затухает, стремясь к асимптотическому нулю. Это свидетельствует о правильно подобранной скорости обучения.
* **Разделяющая линия:** Поскольку один нейрон без скрытых слоев оперирует уравнением $w_1 x_1 + w_2 x_2 + b = 0$, разделяющая граница представляет собой идеальную прямую линию. На графике `plot_decision_boundary` видно, что эта линия безошибочно отсекает красные точки (*Setosa*) от зеленых (*Versicolor*).
* **Метрики:** Модель достигает идеальных показателей на тесте: $\text{Accuracy} = 1.0000$, $\text{F1-score} = 1.0000$.

### 6.2 Многослойный перцептрон на полной выборке

Во втором эксперименте задача усложняется: обучается полная архитектура `[4, 10, 10, 3]` на трех классах с использованием всех четырех признаков. Классы *Versicolor* и *Virginica* не имеют четкой линейной границы и частично пересекаются в пространстве конфигураций.

* **Параметры обучения:** `lr=0.05`, `epochs=3000`. Более низкий `lr` обусловлен усложнением топологии ландшафта функции потерь глубокой сети: высокий шаг мог бы выбить оптимизатор из узких локальных минимумов.
* **Разделяющая поверхность:** Для визуализации работы MLP строится двумерный срез признакового пространства: по осям откладываются параметры лепестков (*petal features*), а параметры чашелистиков (*sepal features*) фиксируются на их средних значениях. Полученная разделяющая поверхность перестает быть линейной. На графиках видны криволинейные зоны («карманы»), которые формируются под воздействием скрытых слоев. Сеть адаптирует форму границы, огибая плотные кластеры данных.
